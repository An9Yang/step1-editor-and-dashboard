## 三、项目编辑页面（核心任务）

> ✅ **这是本次设计的重点**，需要完全自主设计

### 3.1 整体布局结构

| 区域 | 功能说明 |
|------|----------|
| 左侧面板 | 对话区域（聊天气泡）、历史记录入口、输入框 |
| 右侧面板 | 预览区域、工具面板 |
| 顶部工具栏 | 功能按钮（Code/Preview/Design）、路由显示、分享等 |

**需要新增的能力**：左右面板宽度可拖拽调整（当前版本不支持）

### 3.2 左侧对话区域

#### 3.2.1 气泡设计

- 用户消息与 AI 回复需要有明确的视觉区分
- 建议采用左右对齐方式（用户消息右对齐，AI 回复左对齐），符合直觉
- 参考：V0 采用全部左对齐，但左右对齐更符合用户习惯

#### 3.2.2 间距与排版

**当前问题**：间距不合理，整体观感较差

**改进方向**：
- 优化消息之间的间距
- 优化 Markdown 渲染效果
- 参考 Lovable 和 Create.xyz 的排版处理

#### 3.2.3 代码块处理

**核心原则**：Nobody cares about code（没有人关心代码细节）

- 代码内容**默认折叠**，不展开
- 用户可点击展开查看，但不强制展示
- 参考 Lovable 的处理方式

#### 3.2.4 高亮块/引导块

需要设计引导性高亮块，帮助用户明确下一步操作：

| 高亮块类型 | 用途说明 |
|------------|----------|
| TODO 提示 | 提示用户接下来可以做什么 |
| 问题提示 | 当出现问题时，通过高亮块提示用户 |
| 操作建议 | 强烈指引用户进行下一步操作 |

**参考**：Lovable 在对话末尾会有引导性的高亮块

#### 3.2.5 输入框设计

- **位置**：左下角固定
- **参考**：Lovable 的输入框设计范式，整体交互需流畅

### 3.3 顶部工具栏

#### 3.3.1 核心功能按钮

| 按钮 | 功能说明 |
|------|----------|
| Code | 查看/编辑代码 |
| Preview | 预览效果 |
| Design | 设计模式（Design Mode） |

> ⚠️ **注意**：当前版本缺少 Design 按钮，需要补充

#### 3.3.2 路由显示

- **功能**：显示当前所在的路由/页面路径
- **原因**：产品支持多路由，用户需要知道当前在哪个页面
- **位置**：顶部工具栏中间区域
- **当前问题**：现有版本没有路由显示功能，需要新增

#### 3.3.3 分享按钮

- **位置**：顶部工具栏右侧
- 需要强化分享功能，让用户能快速分享作品到社交媒体

### 3.4 版本历史管理

#### 3.4.1 位置设计

- **推荐位置**：左侧（靠近编辑区域）
- **设计逻辑**：用户在左侧编辑，回退版本操作在左侧更顺手
- **当前问题**：Step1 的历史记录放在右边，不合理

#### 3.4.2 版本命名

**当前问题**：版本名不够 human readable（如只显示 v1）

**改进方案**：
- 采用类似 Git Commit 的命名方式
- 格式：`v1 + 版本描述`（如 "v1 - 添加用户登录功能"）
- 可考虑用 AI 自动生成版本描述

#### 3.4.3 交互优化

- 提供快速回退按钮，而非仅展示版本号
- 版本切换需要清晰的确认机制

### 3.5 发布（Publish）功能

#### 3.5.1 简化设计

**当前问题**：Step1 的 Publish 功能显示过多信息（Production/Preview 等），用户不知道点哪个

**改进方案**：
- 简化为单一的 "Publish" 按钮
- 如果没有 Production 环境，只需提示用户 "Public" 即可
- 参考 Lovable 的简洁处理方式

#### 3.5.2 设计原则

- 去掉冗余信息，隐藏用户不需要关心的技术细节
- 走最简单的路径，降低用户认知负担

### 3.6 导出（Export）功能

**当前问题**：只有一个下载图标，用户不清楚功能

**改进方案**：
- 将图标改为带文字的按钮："Export"
- 文字说明比纯图标更清晰

### 3.7 分享功能设计

#### 3.7.1 分享类型

| 分享类型 | 功能说明 |
|----------|----------|
| 分享网站 URL | 分享已发布网站的链接 |
| 分享项目 | 分享项目编辑器链接，支持多人协作 |
| Invite 邀请 | 邀请他人协作编辑项目（可先保留） |

#### 3.7.2 交互设计

- 点击分享按钮展开下拉菜单，选择分享方式
- **核心目标**：让用户能快速把做好的网站分享到社交媒体
- **暂不优先**：生成网页快照辅助分享

### 3.8 设置（Settings）功能

#### 3.8.1 位置问题

- **当前问题**：Settings 按钮位置不合理
- 需要重新设计 Settings 入口位置

#### 3.8.2 内容区分

**当前问题**：个人设置和项目设置混在一起

**改进方案**：
- 区分「个人设置」和「项目设置」
- 项目设置放在编辑器右侧工具面板
- 个人设置单独入口（如头像下拉菜单）

## 六、设计交付物

### 6.1 交付内容

1. **项目编辑页面**完整设计稿（核心）
2. Dashboard 页面设计稿（参考竞品，轻度设计）
3. 各组件的交互说明
4. 设计规范文档（间距、颜色、字体等）

### 6.2 时间节点

| 时间 | 交付内容 |
|------|----------|
| 周四/周五 | 初版设计稿交付，进行评审和反馈 |
| 评审后 | 根据反馈进行修改完善 |

### 6.3 评审标准

Leader 会从以下角度进行评审：

- 是否解决了现有问题
- 是否符合产品定位
- 交互是否清晰易用
- 视觉是否专业、现代

---

## 七、设计原则

1. **简单优先**：走最简单的路径，不要过度设计
2. **用户导向**：始终考虑用户的实际使用场景
3. **参考竞品**：站在巨人肩膀上，但要有自己的特色
4. **快速迭代**：先出 MVP，根据反馈优化
